Role-Based Access Overview

Visible role blocks are static in HTML. JS and backend will enforce actual role permissions later.

  • President: post, audit all, create events, verify/revoke unpaid members.
  • Vice-President: post, audit all, create events, verify/revoke unpaid members.
  • General Secretary: post, audit all, create events, verify/revoke unpaid members.
  • Treasurer: post, manage money tasks, create events, verify/revoke unpaid members.
  • Member: post articles/research/datasets.

Member Actions

  • Post and read articles
  • Share research items
  • Upload and browse datasets

Admin Actions

  • Create events with date and time details
  • Review payment references
  • Revoke unpaid members
  • Audit member activity
